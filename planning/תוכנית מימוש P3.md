# P3 — Identity per Object: Implementation Plan

**Goal:** v2 step 3 — מנגנוני-זיהוי מדורגים פר-אובייקט, טוגל זיהוי-כפילויות-פנימיות
(ברירת-מחדל **כבוי**), וכפתור "הוסף אובייקט" (יעד-Lookup שלא נטען).

**Architecture:** אין שינוי במנוע — `identity.compute_key` כבר מקבל `list[list[str]]`.
P3 = schema + UI בלבד: `IdentityConfig` חדש ב-`config/runtime_schema.py`
(mechanisms מדורגים + dedup_internal) במילון `schema.identity[object_api]`,
ו-`screen_identity()` ב-`main.py` (טאב פר-אובייקט). מאגר-השדות למנגנונים =
שדות ממופים (role=field) מתוך `schema.mappings`; לאובייקט-נוסף (לא-נטען) — כל שדות
המילון שלו. גרירה-לשינוי-סדר נדחית (Streamlit) — הסדר = סדר-הרשימה, מחיקה+הוספה.

## Task 1 — Schema (TDD)
- `IdentityConfig` dataclass: `mechanisms: list[list[str]] = []`, `dedup_internal: bool = False`.
- ב-`RuntimeSchema`: `identity: dict[str, IdentityConfig] = {}`, `extra_objects: list[str] = []`.
- טסטים ב-`tests/test_runtime_schema.py`: ברירות-מחדל (dedup כבוי!), אחסון מנגנונים.
- commit: `feat(v2/P3): IdentityConfig — ranked mechanisms + dedup toggle (off)`

## Task 2 — UI `screen_identity()` ב-`main.py`
- אובייקטים: נטענים (`schema.objects`) + `schema.extra_objects`; טאב לכל אחד
  (extra מסומן "🔍 זיהוי בלבד").
- מאגר-שדות: `[m.field_api for m in mappings if object==obj and role==field and field_api]`
  (ייחודי, בסדר-עמודות); ל-extra — `[f.api for f in dictionary[obj].fields]`.
- פר-טאב: טוגל "זיהוי כפילויות פנימיות" (כבוי), רשימת מנגנונים ממוספרת
  (multiselect צירוף-AND + 🗑️), כפתור "הוסף מנגנון" (עד 5), תקציר חי.
- "➕ הוסף אובייקט": selectbox מאובייקטי-המילון שאינם קיימים → `extra_objects`.
- badge: done ⇔ לכל אובייקט-נטען יש מנגנון לא-ריק אחד לפחות.
- route: `elif step == 3: screen_identity()`.
- AppTest ב-`tests/test_v2_identity_ui.py`: רינדור עם נתונים, ושמירת-מצב.
- commit: `feat(v2/P3): identity screen — per-object tabs, mechanisms, add-object`

## Task 3 — אימות + תיעוד
- `python -m pytest -q` ירוק; אימות חי ב-preview מול "בדיקה v2" + המילון.
- work-log + סימון P3 בספק + commit `chore(v2/P3): P3 complete`.

## הקשר להמשך (P4–P6)
- `schema.identity[obj].mechanisms` מוזן ל-`dedup_engine.deduplicate` (P6) ולפתרון
  Lookups (P4) — "זוהה לפי" ברירת-מחדל = מנגנוני-היעד.
- `dedup_internal=False` ⇒ ב-P6 מדלגים על קיבוץ-פנימי (כל שורה נטענת בנפרד);
  ההצלבה מול ה-DB (insert/upsert) רצה תמיד.
